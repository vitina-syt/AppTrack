# 参赛资料 — StepCast · CAD 操作智能录制与知识沉淀平台

---

## 创意点子描述

**StepCast** 是一款运行在工程师本机的桌面智能体，专为 AutoCAD、PTC Creo 等工业 CAD/CAM 软件设计。它在工程师正常工作时**零干预地在后台自动录制**：通过 COM API 截获每一条 AutoCAD 命令、通过 Windows UI Automation 捕获 Creo 的界面操作、同步录制工程师的语音讲解、并在关键操作节点自动截屏。录制结束后，系统将命令序列 + 语音转写文本 + 截图时序输入 **Claude AI**，自动生成一篇结构清晰的操作解说词；解说词再对接 **HeyGen / D-ID** 数字人平台，输出一段配有真人形象的培训讲解视频——整个过程从"开始录制"到"导出视频"，工程师无需额外编写一个字。

---

## 提案将解决什么业务问题？

### 核心业务问题：工业知识难以从专家头脑中流出

在制造业与工程设计领域，企业的核心竞争力高度依赖于少数资深 CAD 工程师积累的操作经验与设计方法论。这类知识极其难以迁移：

1. **人才流失风险**：资深工程师退休或离职时，数十年积累的 CAD 操作经验随之消失，企业不得不支付高昂的重新培训成本。

2. **培训效率低下**：新员工学习复杂 CAD 操作（如曲面建模、参数化装配约束、三维标注）通常需要 3–6 个月的"师带徒"时间，严重占用高级工程师的生产时间。

3. **标准化作业难以落地**：设计规范与操作流程往往以静态 PDF 或口耳相传的方式存在，无法保证每位工程师按照最优路径操作。

4. **跨国协作障碍**：拥有中国、德国、美国工程团队的企业（如西门子能源等），同一套操作流程需要分别用中文、英文、德文制作培训材料，成本翻倍。

## 当前痛点是什么？

### 痛点一：现有录制工具"录下了画面，丢失了意图"

市面上的通用屏幕录制软件（OBS、Camtasia 等）只能录制像素流，无法理解 CAD 命令的语义。一段"工程师在 AutoCAD 里绘制轴承座"的屏幕录像，没有任何元数据标注哪个操作是 `FILLET`、哪个是 `ARRAY`，后期剪辑和标注全靠人工。

**结果**：制作一段 15 分钟的 CAD 培训视频，资深工程师平均需要额外投入 **4–8 小时**的录制、剪辑、配音、字幕工作。

### 痛点二：AI 工具无法感知 CAD 软件的内部状态

通用 AI 助手（如 ChatGPT、Copilot）能回答 AutoCAD 的理论问题，但无法感知"这位工程师刚刚执行了什么命令序列、他在设计什么零件"。两者之间存在一道**上下文鸿沟**——AI 看不到专家正在做什么。

**结果**：AI 生成的操作说明是通用的、脱离实际操作的，无法替代真实录制内容。

### 痛点三：专业培训视频制作门槛高、成本高

制作一套带数字人讲解的 CAD 培训课程，通常需要：
- 专业录屏 + 剪辑软件（Adobe Premiere 等）
- 配音演员或 TTS 后期处理
- 数字人平台账号与额外制作时间

一套 10 节课的 AutoCAD 培训视频，外包制作报价通常在 **5 万–20 万元**，且内容一旦软件版本更新就需要重新制作。

### 痛点四：知识沉淀缺乏标准化载体

大多数企业的 CAD 知识沉淀仍停留在：口头传授、截图 + Word、偶尔录制的未剪辑视频。这些材料：
- 搜索困难（无命令级别的全文索引）
- 难以维护（一处改动要修改多处文档）
- 无多语言版本

## 可能的 AI 解决方案？

StepCast 提出的解决方案是**端到端的 CAD 知识自动化流水线**，分为四个层次：

### 层次一：多信号同步采集（感知层）

| 信号来源 | 技术手段 | 捕获内容 |
|---------|---------|---------|
| AutoCAD 命令 | COM API（`acad.exe` 进程接口） | 每条命令名称、时间戳、执行上下文 |
| Creo 界面操作 | Windows UI Automation (UIA) | 按钮点击、菜单选择、对话框输入 |
| 工程师语音 | 麦克风录音 + Whisper 转写 | 实时口述说明、操作意图 |
| 屏幕截图 | MSS（多显示器截屏库） | 关键步骤的可视化画面 |
| 前台窗口切换 | Win32 `GetForegroundWindow` | 多软件协同操作的上下文 |

五路信号在同一时间轴上对齐，形成一个**多模态操作事件流**。

### 层次二：AI 理解与解说词生成（理解层）

录制结束后，系统将多模态事件流构造成结构化的 prompt，调用 **Claude API**：

```
输入：
  - AutoCAD 命令序列：LINE → OFFSET → TRIM → FILLET(r=5) → HATCH
  - 语音转写：「先画中心线，偏移 20 毫米，倒圆角半径 5，最后填充剖面线」
  - 截图：step_01.png … step_08.png
  - 会话标题：「绘制轴承座剖视图」

输出：
  一篇结构化的操作解说词，包含：
  - 操作目的概述
  - 逐步骤讲解（命令 → 参数 → 注意事项）
  - 关键技术要点提炼
  - 支持中文 / 英文 / 德文三语输出
```

Claude 的优势在于它能**融合语音描述与命令序列**，补全工程师没有说出口的技术背景，生成远比单纯转写更专业的解说词。

### 层次三：数字人视频生成（呈现层）

生成的解说词文本提交至 **HeyGen 或 D-ID**，配合预设的工程师数字形象，自动合成一段配有：
- 真人形象讲解
- 同步字幕
- 配音（支持中/英/德语音）

的培训视频，无需任何视频剪辑操作。

### 层次四：本地幻灯视频（轻量备选）

对于不需要数字人的场景，系统使用 **ffmpeg / Pillow** 将截图序列合成为带时间戳标注的本地 MP4/GIF 幻灯视频，作为成本更低的培训素材。

## 技术创新点总结

1. **CAD 语义感知**：不同于通用屏幕录制，通过 AutoCAD COM API 直接获取命令语义，而非像素识别，信号质量高、开销低。

2. **多模态融合 prompt**：将命令序列、语音文本、截图时序三路信号融合为一个有时序结构的上下文，引导 Claude 生成专业解说词，而非通用说明。

3. **本地优先架构**：所有数据存储在本机 SQLite，仅 AI 生成和数字人导出时调用外部 API，满足制造业对数据安全的合规要求。

4. **零侵入录制**：无需对 AutoCAD/Creo 进行任何插件安装或配置修改，通过系统级 API 旁路监听，对工程师工作流程零影响。

5. **可扩展 Agent 架构**：CreoScribe（Creo）与 AutoCAD 模块共享同一套数据模型和 API 框架，新增对 SolidWorks、NX、CATIA 的支持只需接入对应的命令捕获适配器。
