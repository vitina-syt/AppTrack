## 执行步骤

### 1. 扫描代码现状

**后端（`backend/app/`）：**
- 读取 `main.py` — 获取 lifespan、中间件、挂载路由
- 读取 `api/routes/` 下所有路由文件 — 汇总全部 API 端点（方法、路径、描述）
- 读取 `models.py` — 了解数据模型
- 读取 `tracker.py` — 了解追踪器逻辑
- 读取 `database.py` — 了解数据库 schema

**前端（`frontend/src/`）：**
- 读取 `App.jsx` — 获取所有路由页面
- 读取 `pages/` 下每个页面文件 — 汇总页面功能
- 读取 `store/` 下 store 文件 — 了解持久化设置项
- 读取 `i18n/locale.js` — 了解支持的语言

**配置：**
- 读取 `backend/requirements.txt` — 后端依赖
- 读取 `frontend/package.json` — 前端依赖和技术栈

### 2. 对比文档

读取 `README.md`，逐节比较：

| README 章节 | 检查点 |
|-------------|--------|
| 功能概览 | 表中是否缺少新功能行 |
| API 接口 | 是否有新端点未列入表格 |
| 前端页面 | 是否有新页面未描述 |
| 设置（`/settings`） | 是否有新设置项未列出 |
| 目录结构 | 是否有新增文件未出现 |
| 技术栈 | 是否引入了新依赖未列入表格 |
| 数据存储 | 数据库 schema 是否有变化 |

### 3. 更新 README.md

对每处差异，**就地编辑** README.md 对应章节：

- 新增 API 端点 → 追加到对应表格
- 新增页面 → 在"前端页面"章节添加小节
- 新增设置项 → 追加到设置页描述列表
- 新增文件 → 在目录结构代码块中补充
- 新增依赖 → 追加到技术栈表格
- schema 变更 → 更新 SQL 代码块
- 新功能 → 在"功能概览"表格追加行

**原则：**
- 保持现有中文行文风格和格式不变
- 只追加或修改，不删除已有内容
- 每次编辑后重读确认格式正确
